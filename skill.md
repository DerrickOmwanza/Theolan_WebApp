# Node.js + React Specialist Skill

**Purpose:** Build and optimize Node.js + React applications with scalable, maintainable architecture.

**Scope:** Applicable to full-stack web apps using Node.js backend and React frontend.

**Output:** Production-ready, performant applications with clean architecture.

---

## PROFESSIONAL STANDARDS
- SOLID principles, Clean Architecture
- React best practices (hooks, state management, component modularity)
- Node.js performance tuning (async, clustering, caching)
- Security: OWASP Top 10, JWT, CSRF/XSS protection
- CI/CD integration

---

## WORKFLOW
1. **Frontend Audit**: Component hierarchy, routing, state management.
2. **Backend Audit**: API endpoints, middleware, error handling.
3. **Performance Review**: Bundle sizes, query optimization, caching.
4. **Security Review**: Input validation, authentication, authorization.
5. **Testing Strategy**: Jest, React Testing Library, Supertest for APIs.

---

## EXECUTION TIPS
- Use ESLint + Prettier for code quality.
- Apply lazy loading and code splitting in React.
- Implement caching (Redis) for Node.js APIs.
- Monitor performance with Lighthouse and k6.

# Automated Testing Strategy Skill

**Purpose:** Plan and implement automated testing across all layers.

**Scope:** Unit, integration, end-to-end, performance, and security testing.

**Output:** CI/CD integrated testing suite with coverage reports.

---

## PROFESSIONAL STANDARDS
- Test Pyramid (unit > integration > E2E)
- Continuous Testing in CI/CD
- Coverage thresholds (80%+)
- Security testing (OWASP ZAP, dependency checks)

---

## WORKFLOW
1. **Unit Testing**: Core functions, models, utilities.
2. **Integration Testing**: API endpoints, database interactions.
3. **End-to-End Testing**: User flows with Cypress/Selenium.
4. **Performance Testing**: Load/stress tests with k6.
5. **Security Testing**: Vulnerability scans in pipeline.

---

## EXECUTION TIPS
- Automate test runs in GitHub Actions/GitLab CI.
- Use mocks/stubs for external APIs.
- Track coverage with Jest/pytest reports.
- Fail builds if coverage drops below threshold.
